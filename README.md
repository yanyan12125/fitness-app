# 我的健身 💪

一个移动端优先的健身训练助手，帮你创建训练计划、跟练计时、记录每次训练。

## 功能

- **训练计划管理** — 创建、编辑、删除自定义训练计划，支持力量训练和有氧运动
- **训练播放器** — 跟练模式，带倒计时、组间休息提醒、进度条
- **语音播报** — 可开启语音指导，解放双手看屏幕
- **有氧计时** — 跑步、骑行、跳绳等有氧训练独立计时
- **训练记录** — 自动保存每次训练数据（时长、动作数、组数），支持回顾和模板复用
- **深色模式** — 支持浅色/深色主题切换

## 技术栈

- 纯前端，单个 HTML 文件，无需后端
- 原生 JavaScript，无框架依赖
- 数据存储在浏览器 localStorage
- 移动端适配（支持 safe-area、触控优化）
- Web Speech API 语音播报

## 一个完整的项目通常还需要什么？

这个项目目前是最精简的状态（单文件），一个更完整的项目结构通常包含：

```
my-project/
├── README.md          ← 项目说明书（你正在看的）
├── index.html         ← 页面结构
├── css/
│   └── style.css      ← 样式（从 HTML 里拆出来）
├── js/
│   └── app.js         ← 逻辑（从 HTML 里拆出来）
├── assets/            ← 图片、图标等资源
├── docs/
│   └── PRD.md         ← 产品需求文档
├── .gitignore         ← 告诉 Git 哪些文件不要上传
├── LICENSE            ← 开源协议
└── package.json       ← 项目依赖和配置（如果用了 npm 包）
```

各部分的作用：

| 文件/文件夹 | 是什么 | 为什么需要 |
|------------|--------|-----------|
| **README.md** | 项目说明书 | 别人打开你的 GitHub 第一眼看到的东西 |
| **PRD.md** | 产品需求文档 | 写清楚这个产品要做什么、不做什么 |
| **.gitignore** | Git 忽略规则 | 避免把不该上传的文件传到 GitHub |
| **LICENSE** | 开源协议 | 告诉别人能不能用你的代码 |
| **package.json** | 依赖配置 | 项目用了哪些第三方库，一键安装 |
| **css/ js/** | 拆分后的代码 | 代码分文件管理，更清晰，多人协作不冲突 |
