今天我们都做了什么？

一个网页从「电脑上的文件」变成「手机上的 App」，每一步背后发生了什么

↓
第一章

HTML 文件是什么？

类比理解

把一个网页想象成一封信。HTML 就是信的内容和格式——标题在哪、正文在哪、图片在哪。浏览器就是帮你读这封信的邮差。

你的健身 App 整个就是一个 index.html 文件。浏览器打开它，按照里面的指令把页面画出来。

每个 HTML 文件都分成两部分：

<head> <!-- 给浏览器/手机看的"说明书"，用户看不见 -->
</head>

<body> <!-- 用户真正看到的页面内容 -->
</body>
💡 关键认知

<head> 里的东西用户完全看不到，但它决定了网页的"身份"——叫什么名字、长什么图标、怎么在手机上显示。我们今天改的大部分东西，都在 head 里。

第二章

Meta 标签：网页的身份证

类比理解

Meta 标签就像你护照上的信息——你自己可能不会天天看，但海关（浏览器）需要它来了解你是谁、从哪来、说什么语言。

来看看你的健身 App 里的每一行 meta 都在做什么：

<meta charset="UTF-8">

作用：告诉浏览器"这个网页用 UTF-8 编码"。

大白话：就像告诉翻译官"这封信是中文写的"。没有这行，中文可能变成乱码 ⬜⬜⬜。

<meta name="viewport" content="width=device-width, initial-scale=1.0">

作用：让网页在手机上正确显示，不要缩成一个小页面。

大白话：手机屏幕和电脑屏幕大小不一样。这行告诉浏览器"请按照手机屏幕的实际宽度来显示，别假装自己是电脑"。没有这行，你的网页在手机上会变得巨小。

<meta name="theme-color" content="#c084fc">

作用：设置浏览器顶部状态栏的颜色。

大白话：打开你的 App 时，手机最顶上那条状态栏（显示时间、电量的地方）会变成紫色，看起来更像一个真正的 App，不像在浏览器里看网页。

<meta name="apple-mobile-web-app-capable" content="yes">
<meta name="apple-mobile-web-app-status-bar-style" content="black-translucent">

作用：专门给 iPhone 看的指令。

第一行：告诉 iPhone"这个网页可以全屏运行"——添加到主屏幕后打开，不会显示浏览器的地址栏和工具栏，看起来就像原生 App。

第二行：让状态栏半透明，和你的 App 背景色融为一体。

第三章

manifest.json：App 的出生证明

类比理解

如果 meta 是护照，那 manifest.json 就是出生证明——它定义了你的 App 叫什么名字、图标长什么样、打开时什么颜色。手机靠这个文件来决定怎么"安装"你的网页。

"name": "我的健身",
"short_name": "健身"

name：App 的完整名字，在安装提示里显示。

short_name：缩写名字，显示在手机桌面图标下方（因为桌面空间小，名字太长会被截断）。

"display": "standalone"

standalone 的意思："独立运行"——打开后没有浏览器的地址栏、前进后退按钮，看起来就是一个独立的 App。

其他选项：browser（在浏览器里打开）、fullscreen（全屏，连状态栏都没有，适合游戏）。

"icons": [{
"src": "icon-180.png",
"sizes": "180x180",
"type": "image/png"
}]

作用：告诉手机"我的图标在哪里、多大、什么格式"。

为什么你之前图标显示的是"我"字？因为没有提供 PNG 图标，iPhone 就自动截取了网页的第一个字作为图标。加了这个之后，它就知道要用你的哑铃图标了。

"background_color": "#faf5ff",
"theme_color": "#c084fc"

background_color：App 启动时、内容还没加载出来之前，屏幕显示的底色。就像 App 的"开屏背景"。

theme_color：手机系统 UI（状态栏等）的颜色，和前面 meta 里的 theme-color 作用类似，但这个是给 Android 用的。

🤔 为什么 meta 和 manifest 有重复的信息？

因为历史原因，iPhone（iOS）和 Android 读的东西不一样。iPhone 主要看 meta 标签和 apple-touch-icon，Android 主要看 manifest.json。所以两边都要写，才能保证所有手机都能正确显示。

第四章

从电脑到全世界：部署是怎么回事

类比理解

你写了一本书（代码），现在放在你家书架上（电脑）。别人想看？他们得来你家。部署就是把书送到图书馆（服务器），任何人随时都能借阅。

1

你的电脑上有代码

index.html 只存在你的硬盘里，地址是 file:///Users/... 开头的，别人访问不了

2

推到 GitHub

代码上传到 GitHub 的服务器，有了备份和版本管理，但还不是网站

3

Netlify 读取 GitHub

Netlify 从 GitHub 拉取你的代码，放到它的服务器上，生成一个 https:// 开头的公开链接

4

全世界都能访问了

任何设备打开那个链接，Netlify 的服务器就把你的文件发给他们的浏览器

✨

自动更新

以后你改了代码推到 GitHub，Netlify 会自动检测到变化，几秒钟内更新线上版本

第五章

全景图：每个文件都在做什么

index.html — 整个 App 的全部内容

包含三个部分：

1. <style> 标签里的 CSS — 决定长什么样（颜色、布局、动画）

2. HTML 标签 — 决定有什么内容（按钮、文字、列表）

3. <script> 标签里的 JavaScript — 决定怎么交互（点击、计时、记录）

manifest.json — App 的身份配置

告诉手机：这个网页可以当 App 安装，它叫什么名字、图标在哪、要不要全屏显示。

没有这个文件，"添加到主屏幕"就只是一个普通的网页快捷方式。有了它，才真正变成 PWA。

icon-180.png — App 图标

180x180 像素的 PNG 图片。为什么是 180？因为 iPhone 主屏幕图标需要这个尺寸（60pt x 3倍屏幕 = 180px）。

之前用 SVG 格式 iPhone 不认，换成 PNG 就好了——这就是为什么了解平台差异很重要。

README.md — 项目说明书

给人看的，不是给机器看的。打开 GitHub 仓库页面时，这个文件的内容会自动显示在首页。

好的 README 让别人 30 秒内知道你的项目是什么、怎么用。也是你作品集的一部分。
